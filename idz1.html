Індивідуальне домашнє завдання

Індивідуальне домашнє завдання (ІДЗ) є одним із видів робіт у рамках курсу Internet-технології (1-й семестр).
Оцінюється виконане ІДЗ на підставі його здачі (окремих понять "відпрацювання" та "захисту" у контексті ІДЗ немає). Складання ІДЗ полягає у демонстрації виконаного завдання, його вихідного коду та відповідях на запитання по ходу перевірки, з обов'язковим подальшим надсиланням посилання на репозиторій.
Вибір варіанта завдання здійснюється на власний розсуд із запропонованих нижче. На відміну від додаткових завдань, узгоджувати цей вибір не потрібно. Обрали, щось уточнили (якщо потрібно), зробили, показали (на лабораторній чи консультації), отримали оцінку.
Крайній термін здачі ІДЗ - день виставлення заліку.
При виконанні завдань пам'ятайте, що гуглом викладач теж користуватися вміє, і несумлінне "повторне використання" призведе до несподіваних цікавих запитань у майбутньому.

Доступні такі теми індивідуальних завдань, на ваш вибір:

JS, Букмарклет, Хабр.
Букмарклет для обробки сторінок статей ресурсу habr.com. Видалення бічної, верхньої та нижньої панелей, кнопки "Вгору" (що виникає під час прокручування), скриптів гугл-аналітики тощо. Тобто, тих елементів сторінки, які не потрібні для відображення її контенту користувачеві. Зверніть увагу на відмінності у структурі сторінок статей звичайних користувачів та корпоративних блогів.
Максимальна оцінка: 7 балів.

JS, Букмарклет, SO.
Букмарклет для обробки сторінок з запитаннями/відповідями на ресурсі stackoverflow.com, приклад типової сторінки, що обробляється. Завдання букмарклета полягає у видаленні непотрібних користувачеві елементів сторінки (футер, лівий сайдбар, зайві скрипти тощо) та виділення (кольором, тлом або т.п.) прийнятої відповіді, якщо вона є.
Максимальна оцінка: 7 балів.

JS, Онлайн-таймер.
Застосунок на JavaScript, розгорнутий на Github Pages, для оповіщення користувача із заданою періодичністю. Відображення часу, що залишився до оповіщення на сторінці та в заголовку вікна. Можливість вказати період роботи, запуск, призупинення, скасування таймера та налаштування дії після завершення відліку (припинення роботи або повторний запуск таймера). Оповіщення через Notification API. Можливість керування таймером не тільки через елементи інтерфейсу на сторінці, але й через параметри GET-запиту (тобто параметри роботи таймера – періодичність та інші – можуть бути вказані в адресному рядку, що відкриває можливості для цікавих сценаріїв використання).
Максимальна оцінка: 10 балів.

JS, Гра, Хрестики-нулики.
Два режими гри: розрахований на двох користувачів і однокористувацький. У розрахованому на двох користувачів гра ведеться двома гравцями в одному застосунку, по черзі. В однокористувацькому буде потрібна реалізація логіки нескладного "бота", що відповідає на ходи єдиного гравця. Передбачити можливості: скидання ігрового поля (початок гри), відображення черговості поточного ходу, ведення лічильника ігор та їх результатів.
Максимальна оцінка: 9 балів.

JS, Гра, 15.
Опис гри доступний за посиланням. Передбачити можливість перемішування кісточок на початку гри, з урахуванням випадків, коли впорядкувати неможливо.
Максимальна оцінка: 9 балів.

JS, Гра, Breakout.
Застосунок на JavaScript з використанням canvas, розгорнутий на Github Pages. Опис гри доступний за посиланням.
Максимальна оцінка: 10 балів.

JS, Гра, Перегони.
Застосунок на JavaScript з використанням canvas, розгорнутий на Github Pages. Двовимірна "нескінченна дорога", якою переміщається автомобіль гравця та автомобілі NPC. Управління переміщенням автомобіля гравця вліво-вправо (для уникнення зіткнення з автомобілями NPC, що випадково з'являються дорогою) здійснюється клавішами клавіатури. Під час зіткнення з NPC гра завершується. Ведення рахунку очок, у якості яких можна використовувати пройдений гравцем "кілометраж" дороги до завершення гри.
Максимальна оцінка: 11 балів.

JS, Гра, Морський бій (з ПК).
Класична гра на JavaScript, у якій опонентом гравця є ПК. Необхідно передбачити поведінку бота при розстановці своїх кораблів, і при відстрілі по кораблям гравця.
Максимальна оцінка: 14 балів.

JS, Гра, ЗмійкаБот.
Застосунок на JavaScript з використанням canvas, розгорнутий на Github Pages. На відміну від класичної гри (гравець керує змійкою на клітковому полі, з метою збору ресурсів (що з'являються у випадкових місцях), зростання змійки та виживання), у цьому завданні необхідно забезпечити "самостійність" змійки шляхом реалізації логіки її переміщення. При оцінюванні буде враховано успішність виживання та зростання змійки.
Максимальна оцінка: 11 балів.

JS, Гра, Життя.
Застосунок на JavaScript, розгорнутий на Github Pages. Ілюстрація роботи клітинного автомата, який полягає у розвитку (зміні стану з часом) множини клітин на полі, причому перебіг розвитку заданий параметрами, встановленими користувачем ще до початку гри. Класика жанру Zero Player Game (з певною наукоємністю, втім, це вже за межами нашого курсу). Опис гри доступний за посиланням. Надати користувачеві можливості встановлення параметрів гри, генерації початкового стану поля гри, зміни стану обраних клітин власноруч, покрокового перегляду ходу гри.
Максимальна оцінка: 12 балів.

JS, шифрування Цезаря.
Завдання полягає у створенні статичного застосунка, що розгорнутий на Github Pages, у якому можна виконувати операція кодування (вказати рядок, який потрібно зашифрувати, вказати ключ та отримати шифровану строку) та декодування (ввести шифрований рядок, вказати ключ, отримати вихідну строку) з використанням шифру Цезаря. Шифр Цезаря є одним з найпростіших (до речі, ще будете його вивчати на відповідному предметі) для реалізації, опис принципу шифрування доступний за посиланням.
Максимальна оцінка: 10 балів.

Статичний веб-ресурс (HTML/CSS/JS), розгорнутий на Github Pages, на довільну тематику (опис гри чи фільму, фан-сайт музичної групи чи футбольної команди, портфоліо своєї команди розробників, меню піцерії, опис міста, програмного забезпечення, навчального посібника, проблем садівництва тощо), створений з використанням інструменту gulp (з плагінами для CSS-препроцесорів та мініфікації). Наявність ТЗ до проекту (що включає опис завдань ресурсу, вказівку цільової аудиторії, існуючі аналоги та їх плюси/мінуси, передбачуваний функціонал та засоби розробки) є обов'язковою. Формат ТЗ довільний, розміщується у файлі README.md (з використанням відповідного синтаксису) у репозиторію з проектом. При перевірці, в тому числі, враховуватиметься відповідність контенту заявленої в ТЗ тематиці та адекватність використовуваних технологій для вирішених завдань.
Максимальна оцінка: 12 балів.

PHP, CLI утиліта.
Завдання на знайомство з використанням PHP-скриптів у консолі (PHP CLI). Утиліта для розподілу файлів фотографій за папками, відповідно до даних EXIF про дату знімка.
Максимальна оцінка: 10 балів.

PHP, Бібліотека.
Можливість додавання текстових (або іншого формату) файлів на ресурс, відображення списку наявних раніше завантажених файлів та їх відкриття.
Максимальна оцінка: 10 балів.

PHP, Блог.
Користувач реєструється (облікові дані зберігаються у файлі на сервері - повноцінні БД поки що використовувати не можна), після чого може додавати записи, що зберігаються на сервері в текстовому файлі, асоційованому саме з цим користувачем. При вході користувача відображаються попередні зроблені ним записи.
Максимальна оцінка: 12 балів.

PHP, Калькулятор витрат.
Можливість реєстрації та авторизації користувача, після авторизації користувачеві відображається список його витрат (раніше внесених ним) та можливе додавання користувачем інформації про свої нові витрати (назва позиції та вартість). Для зберігання облікових даних користувача та інформації про його витрати використовувати файли на сервері (повноцінні БД поки що використовувати не можна), доцільно використовувати окремий файл для зберігання даних про витрати кожного користувача.
Максимальна оцінка: 12 балів.
