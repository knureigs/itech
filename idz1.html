Індивідуальне домашнє завдання

Індивідуальне домашнє завдання (ІДЗ) є одним із видів робіт у рамках курсу.
Оцінюється виконане ІДЗ на підставі його здачі (окремих понять "відпрацювання" та "захисту" у контексті ІДЗ немає). Складання ІДЗ полягає у демонстрації виконаного завдання, його вихідного коду та відповідях на запитання по ходу перевірки, з обов'язковим подальшим надсиланням посилання на репозиторій.
Вибір варіанта завдання здійснюється на власний розсуд із запропонованих нижче. На відміну від додаткових завдань, узгоджувати цей вибір не потрібно. Обрали, щось уточнили у лектора для повного розуміння завдання (якщо вважаєте за необхідне), зробили, показали (лектору на консультації), отримали оцінку.
Крайній термін здачі ІДЗ - день виставлення заліку. Краще, звичайно, раніше.
При виконанні завдань пам'ятайте, що гуглом (а також ChatGPT і цілою бібліотекою реалізацій завдань студентами минулих років) викладач теж користуватися вміє, і несумлінне "повторне використання" ускладнює отримання оцінки за ІДЗ і взагалі заліку по курсу.

Доступні такі теми індивідуальних завдань, на ваш вибір:

JS, Букмарклет, Хабр.
Букмарклет для обробки сторінок статей ресурсу habr.com. Видалення бічної, верхньої та нижньої панелей, кнопки "Вгору" (що виникає під час прокручування), скриптів гугл-аналітики тощо. Тобто, тих елементів сторінки, які не потрібні для відображення її контенту користувачеві. Зверніть увагу на відмінності у структурі сторінок статей звичайних користувачів та корпоративних блогів.
Максимальна оцінка: 7 балів.

JS, Букмарклет, SO.
Букмарклет для обробки сторінок з запитаннями/відповідями на ресурсі stackoverflow.com, приклад типової сторінки, що обробляється. Завдання букмарклета полягає у видаленні непотрібних користувачеві елементів сторінки (футер, лівий сайдбар, зайві скрипти тощо) та виділення (кольором, фоном або т.п.) прийнятої відповіді, якщо вона є.
Максимальна оцінка: 7 балів.

JS, Онлайн-таймер.
Застосунок на JavaScript, розгорнутий на Github Pages, для оповіщення користувача із заданою періодичністю. Відображення часу, що залишився до оповіщення на сторінці та в заголовку вікна. Можливість вказати період роботи, запуск, призупинення, скасування таймера та налаштування дії після завершення відліку (припинення роботи або повторний запуск таймера). Оповіщення через Notification API. Можливість керування таймером не тільки через елементи інтерфейсу на сторінці, але й через параметри GET-запиту (тобто параметри роботи таймера – періодичність та інші – можуть бути вказані в адресному рядку, що відкриває можливості для цікавих сценаріїв використання).
Максимальна оцінка: 9 балів.

Застосунок на JavaScript з використанням canvas, розгорнутий на Github Pages. Двовимірна "нескінченна дорога", якою переміщається автомобіль гравця та автомобілі NPC. Класика жанру Zero Player Game (з певною наукоємністю, втім, це вже за межами нашого курсу). Опис гри доступний за посиланням. Надати користувачеві можливості встановлення параметрів гри, генерації початкового стану поля гри, зміни стану обраних клітин власноруч, покрокового перегляду ходу гри.

Шифр Цезаря є одним з найпростіших (до речі, ще будете його вивчати на відповідному предметі) для реалізації, опис принципу шифрування доступний за посиланням.

JS, Відпустка.
Веб-застосунок, розгорнутий на GitHub Pages, що дозволяє користувачу розрахувати невідомий параметр відпустки. Усього параметрів відпустки три – дата початку, тривалість відпустки (число днів), дата завершення. Дати початку та завершення рахуються включно (за замовчуванням).
Застосунок повинен визначати значення будь-якого з трьох параметрів відпустки при відомих значеннях двох інших. Тобто, якщо відомі дати початку та завершення відпустки – розраховується тривалість. Якщо відома дата закінчення та тривалість – застосунок визначає дату початку. Якщо користувач ввів дату початку та тривалість – то розрахувати дату закінчення відпустки.
Необхідна перевірка на те, що дата початку або завершення попадає на неділю (просто попереджати користувача у такому випадку) та можливість вказати дати святкових днів, які не будуть рахуватися у відпустку при розрахунках.
Максимальна оцінка: 11 балів.

JS, Розклад платежів.
Веб-застосунок, розгорнутий на GitHub Pages, для розрахунку дати наступної оплати певної послуги. Користувач вказує поточну суму на рахунку, поточну дату (за замовчуванням – застосунок використовує реальну поточну дату), місячний тариф, дата списання місячного тарифу з рахунку (за замовчуванням – в останній день місяця).
Після вказівки всіх необхідних даних, програма повідомляє, до якого числа необхідно буде поповнити рахунок на мінімальну суму (місячний тариф).
Наприклад: поточна сума на рахунку 655 грн, поточна дата 2025.10.23, місячний тариф 300 грн, тариф списується 1 числа. По розрахункам застосунку виходить, що потрібно поповнити буде до 2026.01.01.
Максимальна оцінка: 11 балів.

При перевірці, в тому числі, враховуватиметься відповідність контенту заявленої в ТЗ тематиці та адекватність використовуваних технологій для вирішених завдань.
