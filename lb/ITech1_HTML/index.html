Лабораторна робота №1

Основи побудови та публікації HTML-документів
1.1 Мета роботи
  1. Вивчити принципи побудови HTML-документа, його структуру та форматування. Розглянути можливості форматування тексту, організації списків, використання гіперпосилань, форм та графіки у web-документах. Поняття табличного подання даних.
  2. Ознайомитись із системою контролю версій (CКВ) Git та можливостями сервісу Github Pages.
1.2 Необхідні засоби
Для виконання лабораторної роботи рекомендується використовувати таке відкрите програмне забезпечення:
  • редактор коду MS Visual Studio Code або будь-який інший, який знайомий користувачу і забезпечує підсвічування синтаксису (Notepad++, Sublime Text, Atom, Brackets и т.п.);
  • браузер Mozilla Firefox або Google Chrome;
  • СКВ Git (для роботи з сервісом GitHub Pages потрібен доступ до мережі Інтернет).
1.3 Порядок виконання роботи
  1. Створити HTML-документ на довільну тематику на основі довільного макета сторінки (одноколонкового, 2-х або 3-х колонкового), приклад структури сторінки з двома колонками). Основний контент сторінки повинен містити блокові та рядкові елементи, заголовки. Зміст довільний. Можна використовувати табличну верстку ‐ з метою кращого знайомства з HTML та розуміння недоліків цього способу побудови сторінки. Загалом дану лабораторну роботу потрібно виконувати без використання CSS. Застосування семантичної верстки замість табличної (за бажанням!), у разі простої одноколонкової структури сторінки, можливо і без управління розміщенням блоків засобами CSS.
  2. Відповідно до варіанту, обов'язково використовувати у створеному документі:
    • фрагмент оформленого тексту (увага на вказане у варіанті його зображення), використовуючи теги для напівжирного виділення тексту, виділення курсивом, під- та надрядкового креслення, а також використовуючи &-сутності для спеціальних символів (стрілок, символів грецького алфавіту, кутових дужок і т.п.);
    • список;
    • таблицю;
    • форму с керуючими елементами;
    • гіперпосилання та зображення.
  3. Скористатися сервісом Github Pages (див. вказівки ‐ з моменту їх створення інтерфейс гітхабу оновився, але самі налаштування та їх порядок незмінні). Для роботи з локальним репозиторієм та взаємодії з віддаленим репозиторієм на Github використовувати консоль (cmd, PowerShell, Git Bash або їх аналоги, що дозволяють без підказок використовувати команди Git). При отриманні відпрацювання можливі запитання стосовно роботи з репозиторієм.
  4. Перевірити відображення створеного веб-ресурсу у різних браузерах.
  5. Перевірити створений документ на валідність за допомогою онлайн-сервису, в якому використовувати URL-адресу опублікованого на Github Pages документа.
  6. Внести до проекту вказані викладачем (при спробі отримання відпрацювання) зміни.
Завдання для лабораторної роботи є індивідуальними. Можливе спільне виконання у бригаді варіантів усіх її учасників.
Номер варіанта студента вказано в електронному журналі. Виконання чужого варіанту є підставою для відхилення роботи.
Відпрацювання виставляється після спільної зі студентом перевірки коректності роботи проекту, відповідей студентом на запитання щодо ходу роботи та (опціонально) отримання викладачем посилання на GitHub репозиторій проекту.
1.4 Варіанти завдань
Варіант Оформлений текст Список Таблиця Форма с керуючими елементами Посилання та зображення
0 Дворівневий список з питаннями та варіантами відповідей. Таблиця з даними про успішність 3-х студентів з деяких дисциплін 3-го курсу. Форма реєстрації відвідувача сайту. Зображення-банер, що веде на офіційний сайт університету.
1 Багаторівневий список, що відображає структуру університету (поділ по факультетам та кафедрам). Таблиця з даними курсу валют. Форма голосування. Зміст сторінки, що складається з відносних гіперпосилань-якорів до її розділів.
2 Дворівневий список, що показує розподіл груп за спеціальностями (СП, КСМ...) або траекторіями (перша, друга...). Таблиця з даними прогнозу погоди. Форма для пошуку. Відносне гіперпосилання для переходу до іншої сторінки цього ресурсу.
3 Випадаючий список студентів групи (можна обмежитися студентами однієї бригади). Таблиця з даними змісту довільної книги (дві колонки, з назвою розділів та відповідними сторінками). Форма для введення інформації про студента. Відносне гіперпосилання, що веде до необхідного розділу іншої сторінки цього ресурсу.
4 Список дисциплін 3-го курсу, створений за допомогою списку визначень. Фрагмент (довільного обсягу) табличного представлення періодичної системи хімічних елементів. Форма для введення інформації про комплектуючі ПК. Завдання на використання карт зображень (тегів map і area). Зображення, по клікам на яке відбувається перехід на сайт університету, сайт кафедри або розклад ‐ в залежності від місця кліка по зображенню.
1.5 Зміст звіту
Звіт з лабораторної роботи повинен містити: мету роботи, варіант завдання, ескіз макета сторінки, вихідний код, зображення отриманого документа, зображення результатів валідації, висновки по роботі. Наявність звіту не є обов'язковою.
1.6 Контрольні запитання
  1. Назвіть основні елементи HTML-документа. Якими тегами визначається його структура?
  2. Як закоментувати фрагмент HTML-коду?
  3. Як у мові HTML позначаються спеціальні сутності (такі, як символи <, & чи >)?
  4. Призначення елемента <!DOCTYPE>.
  5. Теги секції заголовка.
  6. Призначення метатегів.
  7. Як, використовуючи метатеги, заборонити індексацію сторінки пошуковими роботами?
  8. Атрибути тега link.
  9. Як задати сайту Favicon?
  10. Скільки рівнів заголовків має HTML?
  11. Чи можна задавати користувацькі атрибути у HTML?
  12. У чому різниця між тегами <br/> та <BR>?
  13. Наведіть приклади тегів для форматування тексту (курсив, жирний і т.д.).
  14. У чому різниця між елементами <div> і <p>?
  15. У чому різниця між блоковими та рядковими елементами?
  16. Що таке рядково-блокові елементи?
  17. Які теги використовуються для семантичної верстки?
  18. Наведіть приклад створення маркованих, нумерованих списків та списків визначень.
  19. Як змінити вигляд нумерації списку?
  20. Які теги використовуються для створення таблиць?
  21. Як об'єднати комірки таблиці по горизонталі?
  22. Наведіть приклад абсолютного та відносного гіперпосилань.
  23. Які атрибути тега a є обов'язковими (без яких валідацію не буде пройдено)?
  24. Як здійснити перехід до певних розділів інших документів? Як перейти в межах поточного документа?
  25. Як вказати відкриття гіперпосилання у новій вкладці?
  26. Як за допомогою мови HTML можна створити гіперпосилання, яке при натисканні замість переходу пропонує надіслати E-Mail, використовуючи поштовий клієнт користувача?
  27. Які види комп'ютерної графіки ви знаєте?
  28. Чим відрізняється векторна графіка від растрової? У яких випадках доцільно застосовувати растрову графіку, а яких векторну?
  29. Формати графічних файлів для векторної та растрової графіки.
  30. Які особливості графічних форматів GIF, JPEG, PNG?
  31. Які існують способи додати на сторінку SVG-зображення?
  32. Які атрибути тега img є обов'язковими (без яких валідація не буде пройдена)?
  33. Що визначає атрибут decoding для елемента <img>?
  34. Що таке карта зображень? За допомогою яких тегів вона створюється?
  35. У чому полягає схема data:URI для додавання зображень на сторінку?
  36. Для чого використовується тег form? Наведіть приклади керуючих елементів.
  37. Відмінності використання елементів <input type="button"> и <button> для створення кнопки.
  38. Що таке фрейми? Який тег використовується для створення фрейму?
  39. Як ініціалізувати локальний репозиторій у СКВ Git?
  40. Як дізнатися про поточне становище локального репозиторію?
  41. Для чого використовується команда git add?
  42. Як відправити версовані зміни на віддалений репозиторій?
