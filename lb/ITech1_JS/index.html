Лабораторна робота №3

Використання DOM засобами мови JavaScript
3.1 Мета роботи
  • Познайомитись із мовою програмування JavaScript для створення інтерактивних HTML-документів.
  • Розглянути застосування бібліотеки jQuery та засобів Vue.js.
3.2 Необхідні засоби
Для виконання лабораторної роботи рекомендується використовувати таке відкрите програмне забезпечення:
  • редактор коду MS Visual Studio Code або будь-який інший, який знайомий користувачу і забезпечує підсвічування синтаксису (Notepad++, Sublime Text, Atom, Brackets и т.п.);
  • браузер Mozilla Firefox або Google Chrome;
  • СКВ Git (для роботи з сервісом GitHub потрібен доступ до мережі Інтернет).
3.3 Порядок виконання роботи
  1. Створити HTML-документ із довільним контентом (якщо за завданням не передбачається написання букмарклету для заданого web-ресурсу).
  2. Додати скрипт (або підготувати букмарклет) відповідно до варіанту. Можливе застосування бібліотеки jQuery або засобів Vue.js.
  3. Бути готовим до запитань стосовно вашого коду.
  4. Перевірити відображення створеного документа у різних браузерах.
  5. Внести до проекту вказані викладачем (при спробі отримання відпрацювання) зміни.

Лабораторна робота виконується індивідуально, згідно свого варіанту. Номер варіанта студента вказано в електронному журналі. Виконання чужого варіанту є підставою для відхилення роботи.
Відпрацювання може бути отримано шляхом демонстрації студентом повністю виконаної роботи викладачу, з відповідями на запитання викладача щодо ходу роботи та використаних засобів. Можливий запит щодо отримання викладачем посилання на GitHub репозиторій проекту з кодом лабораторної роботи.
3.4 Варіанти завдань

  1. У каталозі images ресурсу знаходяться зображення 1.jpg, 2.jpg, 3.jpg, 4.jpg, 5.jpg, 6.jpg, 7.jpg, 8.jpg, 9.jpg (це для прикладу - просто має бути деяка множина зображень). На сторінці виводиться перше зображення з цієї папки і наявний список, що містить відповідні малюнкам пункти, що дозволяє відобразити малюнок згідно вибору користувача. Надати користувачеві можливість змінювати параметри зображення (ширину та висоту зображення, товщину рамки, альтернативний текст).

  2. Є сторінка з даними про температуру. Написати сценарій (можна у вигляді букмарклету або у вигляді сценарію у складі запропонованої сторінки), що виводить інформацію про найбільш спекотний день (його дату і відповідну температуру).

  3. На сторінці представлена галерея малюнків (декілька зображень, розміщених вертикально) та кнопки "вишче" і "нижче" для зміни їх порядку (при натисканні на ці кнопки зміщується вибране зображення). Спосіб вибору зображення - на власний розсуд. Надати користувачеві можливість змінювати параметри зображення (ширину та висоту зображення, товщину рамки, альтернативний текст).

  4. Реалізувати зміну зображень на сторінці через заданий користувачем (за допомогою модального вікна під час завантаження сторінки) інтервал. Оформлення сторінки (наприклад, фон) повинне змінюватись в залежності від поточного часу доби.
    Написати сценарій для створення таблиці 10 на 10, заповненої згенерованими випадковими цілими числами в діапазоні, вказаному користувачем у полях для введення числових значень. Оформлення таблиці (фон комірок) на кшталт шахової дошки.

  5. Написати сценарій, що випадково генерує дані і виводить їх користувачу у вигляді таблиці, в якій поступово додаються нові рядки. Повинне відрізнятися оформлення парних та непарних рядків даних, а також першого рядка таблиці. Перед виведенням таблиці, користувач повинен вказати інтервал, з яким з'являються нові рядки з даними. Надати можливість зупиняти генерацію даних.

  6. Застосунок, що передбачає вказівку даних студентів (оцінки з предметів) та їх аналіз (визначення числа студентів у кожній з категорій: "відмінник", "хорошист" та "неуспішний"). У формі (анкеті) вказується прізвище та п'ять оцінок за дисципліни (від 2 до 5). "Відмінник" ‐ якщо всі оцінки "5", "хорошист" ‐ оцінки "5" та "4", "неуспішний" ‐ хоча б одна оцінка "2". Куди віднести студента, що має оцінки "3", вирішуйте самостійно. Необхідно послідовно вказати дані для довільної кількості студентів (послідовне заповнення форми даними для різних студентів ‐ внесли для одного, врахували, внесли для наступного, врахували, і т.д.), після чого вивести зведення про кількість студентів у кожній з категорій та перерахувати прізвища неуспішних. Для спрощення налагодження рекомендується (але для відпрацювання не є обов'язковим!) додати можливість не заповнювати вручну оцінки, а використовувати генератор випадкових чисел.

  7. На сторінці знаходиться набір полів для введення даних про п'ятьох співробітників ‐ прізвища та дата прийому на роботу. Розрахувати та вивести користувачеві середній стаж роботи на поточний час, у вигляді цілого числа років.

  8. Калькулятор, що представляє собою набір кнопок для введення цифр та обрання операції (додавання, віднімання, множення, поділ), поле для виразу, що обчислюється (формується шляхом натискання на відповідні кнопки на сторінці), поле для виведення результату.

3.5 Зміст звіту
Звіт з лабораторної роботи повинен містити: мету роботи, варіант завдання, вихідний код web-сторінки, стильових файлів та файлів скриптів, зображення отриманого документа, висновки по роботі. Наявність звіту не є обов'язковою.
