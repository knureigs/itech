1. Мета роботи
  • Вивчення програмних механізмів технології AJAX для динамічної взаємодії з сервером.
  • Знайомство з можливостями Fetch API.
  • Опанування обробки різних форматів відповіді від серверу (text, XML, JSON).

2. Методичні вказівки

Під час підготовки до виконання лабораторної роботи слід ознайомитися з особливостями синхронного та асинхронного методів передачі даних, вивчити формати обміну даними JSON та XML, методи та властивості класу XMLHttpRequest, а також можливості Fetch API.

3. Необхідні засоби

Для виконання лабораторної роботи рекомендується використовувати таке відкрите програмне забезпечення:

  • редактор коду MS Visual Studio Code або будь-який інший, який знайомий користувачу і забезпечує підсвічування синтаксису (Notepad++, Sublime Text, Atom, Brackets и т.п.);
  • збірка для веб-розробки OpenServer;
  • браузер Mozilla Firefox або Google Chrome;
  • вихідний код з лабораторної, що присвячена роботі з даними за допомогою інтерфейсу PDO;
  • СКВ Git (для роботи з сервісом GitHub потрібен доступ до мережі Інтернет).
4. Порядок виконання роботи
  1. Використовується код для роботи з БД із лабораторної роботи, присвяченої роботі з даними за допомогою інтерфейсу PDO. Перевірити його роботу.
  2. Додати на стартову сторінку код, що використовує XMLHttpRequest для відправки запиту до обробників. Змінити код обробників запитів від клієнту таким чином, щоб виведення результату користувачеві не призводило до перезавантаження сторінки - тобто, використовувати технологію Ajax. Використовувати такі формати відповіді від сервера:
    • у форматі простого тексту (безпосередньо отримувати на клієнті згенерований фрагмент HTML-коду та виводити його в заданому елементі стартової сторінки);
    • у форматі XML (результат запиту на вибірку міститься у генерованій сервером XML-відповіді, яку клієнт зчитує через властивість responseXml і формує висновок користувачеві);
    • в формате JSON (результат запиту на вибірку поміщається в масив, який потім перетворюється на JSON-рядок за допомогою методу json_encode і відправляється клієнту, який отримує дані вибірки з рядка за допомогою методу JSON.parse і формує виведення користувачеві).
    Зверніть увагу, необхідно показати застосування всіх трьох форматів відповіді від сервера – по одному формату на кожен із трьох запитів на вибірку.
    Можливе використання замість XMLHttpRequest методів Fetch API для запиту, що є прикладом роботи з форматом JSON. Це може допомогти і на подальшому захисті лабораторної роботи.
5. Зміст звіту

Звіт з лабораторної роботи повинен містити: мету роботи, варіант завдання (тієї лабораторної, яка була основою для даної), вихідний код веб-сторінки (HTML, JavaScript и РНР), зображення отриманого документа при різних запитах, висновки по роботі. Наявність звіту не є обов'язковою. При викладанні вихідного коду на GitHub пам'ятайте про дамп БД.
